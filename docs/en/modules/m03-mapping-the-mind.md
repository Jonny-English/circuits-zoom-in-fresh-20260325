# M03 Mapping the Mind of a Large Language Model

## Goal

Move from “features exist” to “many features can be cataloged, grouped, and browsed.”

## Core reference

- [Mapping the Mind of a Large Language Model](https://www.anthropic.com/research/mapping-mind-language-model)

## What to look for

- Feature discovery becomes useful once you can organize many features into a map.
- A catalog is not just a list of neurons; it is a searchable index of recurring behaviors.
- Coverage matters: discovering many features changes how you think about model analysis.

## Notebook and artifacts

- Notebook: `notebooks/en/m03_mapping_the_mind.ipynb`
- Shared artifact: `artifacts/m03_feature_catalog.json`

## Self-check questions

- Why does a feature catalog change interpretability from isolated anecdotes into something closer to map-making?
- If you had to extend the teaching catalog, which class of features would you add first, and why?
- What stronger research claims are still unsupported even if you can show many feature cards?

## Takeaway

The paper shifts interpretability from isolated anecdotes toward a broader feature atlas.
