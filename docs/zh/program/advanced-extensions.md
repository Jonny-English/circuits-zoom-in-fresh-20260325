# 扩展论文轨道

主线课程故意围绕 Anthropic interpretability 弧线来组织，这样足够清晰。但如果你的目标是进入公司研究，主线之后还需要一条更宽的 extension track。

## 这条轨道为什么重要

- 公司里的 mechanistic interpretability 不会只围绕一家公司的一条论文线展开。
- 你需要补上更通用的 transformer circuits 语言、经典行为电路案例和 auditing 视角。
- 扩展轨道的目标不是全部复现，而是把你的研究视野从“这条主线”扩大到“这个领域常见的问题形状”。

## 使用方法

- 不要一口气全读。每次主线推进到一个阶段，再补一篇对应的扩展论文。
- 每篇都至少交一个输出：framework brief、evidence-chain note、auditing memo 之类。
- 如果时间有限，优先选能把你当前主线模块重新解释一遍的那篇，而不是最火的那篇。

## 推荐顺序

1. `X01` 先补 transformer circuits 总语言。
2. `X02` 看 induction heads，把 attention 图变成具体机制。
3. `X03` 看 IOI，把电路分析推进到自然语言行为。
4. `X04` 看 hidden objectives，把 interpretability 与 auditing 接起来。

## 交付标准

- 不是“我看过了”。
- 是“我能用这篇重新解释主线里的一个模块，或者提出一个新的研究问题”。
