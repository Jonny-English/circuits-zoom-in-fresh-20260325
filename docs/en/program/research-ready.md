# Research-Ready Path

This repository now has two layers:

- an article-by-article interpretability course
- a research-apprenticeship layer designed to make a beginner useful in a company research environment

The second layer matters because the gap between "I followed the notebook" and "I can contribute to a research team" is usually not raw theory. The gap is output discipline.

## What this path is trying to produce

By the end of the 12-week path, a learner should be able to:

- read a paper quickly and explain the real question, method, evidence, limitations, and next experiment
- reproduce a result cleanly and explain which change actually moved the output
- distinguish observation, inference, and speculation in their own writeups
- write short memos that a research engineer or manager could actually scan
- propose a next experiment with a baseline, budget, and stop condition

That is the realistic bar for an entry-level research contributor. It is not a promise that the learner is now a senior researcher. It is a claim that the learner now behaves more like a junior research engineer than like a course consumer.

## Entry requirements

Use this path only if you already have:

- working Python, NumPy, and PyTorch basics
- the ability to run notebooks locally or in Colab
- basic comfort with matrices, norms, logits, and cross-entropy
- time for 8-12 focused hours per week
- willingness to write logs and notes instead of only reading

If any of those are missing, fix them first. This path is dense by design.

## What changes when you switch from "course mode" to "research mode"

In course mode, it is enough to understand the idea.

In research mode, every article should leave behind four artifacts:

1. A paper reading note
2. An experiment log
3. A short memo
4. A next-question list

If you cannot point to these outputs, you are probably still browsing rather than training.

## Non-negotiable operating rules

- Change one variable family at a time.
- Always write down the baseline before you run the variant.
- Always separate observation from interpretation.
- Always ask what would falsify your favorite story.
- Always write one "what this artifact does not prove" paragraph.
- Review weekly, not only at the end.

These rules matter because beginners tend to over-interpret pretty plots and under-document failure.

## Exit portfolio

Finishing the path should leave you with a portfolio, not just a memory:

- 6 or more paper reading notes
- 6 or more experiment logs
- 3 or more short research memos
- 1 graph walkthrough or artifact critique
- 1 two-week capstone proposal

This portfolio is what lets you show research readiness to a mentor, manager, or hiring loop.

## What this path still cannot replace

No repository can literally guarantee company placement. Real research teams still differ in taste, tooling, and domain focus. What this path can do is remove the most common blockers:

- weak paper reading
- weak experiment logging
- weak failure analysis
- weak memo writing
- weak next-step design

If you finish the path seriously, those blockers should no longer be the first thing that gets noticed.
